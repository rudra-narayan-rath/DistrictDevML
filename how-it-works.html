Data & model

What the system analyzes

The prototype uses a curated dataset and a Random Forest model to estimate development priorities.

District indicators

  • Water supply coverage and service quality metrics.
  • Healthcare facility density and accessibility indicators.
  • Road connectivity and infrastructure quality metrics.
  • Education outcomes and infrastructure availability.

Indicators are normalized and combined into a structured input for the ML model.

Machine learning model

  • Random Forest model trained on historical development patterns.
  • Outputs a priority score for each sector per district.
  • Higher scores correspond to more severe development gaps.
  • Scores are exported into a ranked CSV for the web interface.

The prototype focuses on interpretability and uses feature importance to aid explanation.

Budget logic

How budget-aware selection works

The web interface applies a simple optimization routine to select feasible interventions within a fixed budget.

Greedy selection by priority

Interventions are first sorted by their ML priority score for the selected district, highest first.

The system then walks down this ranked list and adds each intervention if its cost still fits inside the remaining budget, stopping when no further items can be added.

  • Input: district-level ranked interventions + user-defined budget.
  • Cost: estimated per sector using prototype cost assumptions.
  • Output: selected interventions, total used budget, and remaining budget.

Transparency

Interpretation and limitations

The prototype is designed for exploration and learning, not as an official planning tool.

What the tool shows

  • Relative priority ranking across major sectors for each district.
  • Illustrative cost assumptions per sector and district.
  • Feasible combinations of actions under a user-entered budget.

What it is not

  • Not a replacement for field surveys or expert judgement.
  • Not an official policy or fund allocation tool.
  • Not a live system; it uses a static, academic dataset.

Outputs should be interpreted as decision-support signals rather than definitive recommendations.
